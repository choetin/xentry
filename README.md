# Xentry

A self-hosted error monitoring and performance analysis platform, similar to Sentry. Designed for desktop and mobile applications that integrate Crashpad.

## Features

- **Crash Monitoring** — Ingests Crashpad minidumps and JSON events, automatically groups and aggregates them by stack fingerprint
- **Symbolication** — Supports Breakpad .sym symbol files for all platforms (Linux/macOS/Windows), uses dump_syms to generate .sym files and minidump-stackwalk to resolve crash stacks
- **Performance Monitoring (APM)** — Ingests trace/span data with transaction lists, waterfall views, and latency statistics
- **Log Management** — Structured log collection with SQLite FTS5 full-text search and log-level filtering
- **Release Management** — Version tracking with per-release crash/event statistics
- **Multi-project Support** — Organization → Project → Environment hierarchy with independent DSN and API Tokens

## Tech Stack

Go 1.22+ / Chi v5 / html/template + HTMX / SQLite (modernc.org/sqlite) / JWT / bcrypt / Tailwind CSS (CDN)

No CGO dependencies — single binary deployment.

## Quick Start

### Binary

```bash
# Build
go build -o xentry ./cmd/xentry

# Run (default listen address :8080)
./xentry

# Or with custom configuration
XENTRY_ADDR=:9090 XENTRY_JWT_SECRET=my-secret ./xentry
```

### Docker

```bash
docker compose up -d
# Visit http://localhost:8080
```

| Variable | Default | Description |

| `XENTRY_ADDR` | `:8080` | Listen address |
| `XENTRY_DB` | `./xentry.db` | SQLite database path |
| `XENTRY_JWT_SECRET` | `change-me-in-production` | JWT signing secret |
| `XENTRY_ENV` | `development` | Runtime environment |

## Usage Guide

### 1. Register and Login

Open `http://localhost:8080/login` to create an account, then sign in with your email and password.

### 2. Create Organization and Project

After logging in, create an organization via the API (you'll need the JWT token from the login response):

```bash
TOKEN="your_jwt_token"

# Create an organization
curl -X POST http://localhost:8080/api/organizations \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Team","slug":"my-team"}'

# Create a project under the organization
curl -X POST http://localhost:8080/api/organizations/{orgID}/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My App","slug":"my-app","platform":"windows"}'
```

The response includes a `dsn_token` that clients use to report data.

### 3. Report Crash Events

Use the project's `dsn_token` to report events via the `X-Xentry-DSN` header:

```bash
DSN="your_dsn_token"

curl -X POST http://localhost:8080/api/proj-123/events \
  -H "X-Xentry-DSN: $DSN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Null pointer dereference",
    "level": "fatal",
    "platform": "windows",
    "release": "1.0.0",
    "threads": [{
      "name": "main",
      "crashed": true,
      "frames": [
        {"function": "main", "file": "main.cpp", "line": 10},
        {"function": "crash_func", "file": "crash.cpp", "line": 42}
      ]
    }]
  }'
```

Upload a minidump file:

```bash
curl -X POST http://localhost:8080/api/proj-123/crash \
  -H "X-Xentry-DSN: $DSN" \
  -F "upload_file_minidump=@crash.dmp" \
  -F "platform=windows" \
  -F "release=1.0.0"
```

Identical stack patterns are automatically aggregated into the same Issue. Duplicate reports only increment the count.

### 4. View Crashes

```bash
# Get all issues for a project
curl http://localhost:8080/api/proj-123/issues \
  -H "X-Xentry-DSN: $DSN"

# Get a single issue's details (including stack frames)
curl http://localhost:8080/api/proj-123/issues/{issueID} \
  -H "X-Xentry-DSN: $DSN"
```

Web UI: visit `http://localhost:8080/projects/proj-123/issues`.

### 5. Upload Symbol Files

Symbol files are used to resolve memory addresses into function names, file names, and line numbers.

```bash
curl -X POST http://localhost:8080/api/proj-123/symbols \
  -H "X-Xentry-DSN: $DSN" \
  -F "file=@app.sym" \
  -F "debug_id=ABC123" \
  -F "type=breakpad" \
  -F "release=1.0.0"
```

Supported symbol formats:

- `breakpad` — Breakpad .sym files generated by dump_syms, resolved via minidump-stackwalk

After uploading, newly reported crash events will automatically trigger asynchronous symbolication.

### 6. Performance Monitoring (APM)

Report trace/span data:

```bash
curl -X POST http://localhost:8080/api/proj-123/traces \
  -H "X-Xentry-DSN: $DSN" \
  -H "Content-Type: application/json" \
  -d '{
    "transactions": [{
      "name": "GET /api/users",
      "trace_id": "trace-abc",
      "span_id": "span-def",
      "op": "http.server",
      "status": "ok",
      "duration": 150.5
    }],
    "spans": [
      {"op": "db.query", "description": "SELECT * FROM users", "duration": 50.0},
      {"op": "http.client", "description": "GET /api/data", "duration": 80.0}
    ]
  }'
```

Query transactions and statistics:

```bash
# Transaction list
curl "http://localhost:8080/api/proj-123/transactions" -H "X-Xentry-DSN: $DSN"

# Statistics (total count, average duration, error count)
curl "http://localhost:8080/api/proj-123/transactions/stats" -H "X-Xentry-DSN: $DSN"

# Transaction details and spans
curl "http://localhost:8080/api/proj-123/transactions/{txID}" -H "X-Xentry-DSN: $DSN"
curl "http://localhost:8080/api/proj-123/transactions/{txID}/spans" -H "X-Xentry-DSN: $DSN"
```

### 7. Log Management

Batch report logs:

```bash
curl -X POST http://localhost:8080/api/proj-123/logs \
  -H "X-Xentry-DSN: $DSN" \
  -H "Content-Type: application/json" \
  -d '[{
    "level": "info",
    "message": "Server started on port 8080",
    "logger": "main",
    "trace_id": "trace-abc"
  }, {
    "level": "error",
    "message": "Connection refused",
    "logger": "db"
  }]'
```

Query and search:

```bash
# Query logs (supports level filtering and pagination)
curl "http://localhost:8080/api/proj-123/logs?level=error&limit=20" -H "X-Xentry-DSN: $DSN"

# Full-text search
curl "http://localhost:8080/api/proj-123/logs/search?q=connection" -H "X-Xentry-DSN: $DSN"
```

### 8. Release Management

```bash
# Create a release
curl -X POST http://localhost:8080/api/proj-123/releases \
  -H "X-Xentry-DSN: $DSN" \
  -H "Content-Type: application/json" \
  -d '{"version":"1.0.0","environment":"production"}'

# List releases
curl http://localhost:8080/api/proj-123/releases -H "X-Xentry-DSN: $DSN"
```

## Crashpad Client Integration

Configure Crashpad in your client code to use Xentry's DSN:

```cpp
#include "client/crashpad_client.h"

// Initialize Crashpad pointing to the Xentry server
crashpad::CrashpadClient client;
client.StartCrashpad("http://your-xentry-server:8080/api/{projectID}/crash",
                    /* annotations */);
```

The client will automatically capture crashes and upload minidumps to the Xentry server.
